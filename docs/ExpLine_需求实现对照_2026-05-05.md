- [x] 已具备核心记录闭环：包装运行实验 -> 捕获 Git 变化 -> 读取项目说明和父实验 -> AI 生成语义记录 -> 保存实验节点 -> 更新索引。
- [x] 已具备基本跨分支父实验比较能力。
- [x] 已具备从实验事实记录重建 `index.json` 的基础能力。

## 部分完成或仍有风险

- [ ] `.expline/` 同步策略尚未明确区分工具仓库与用户实验项目。
  当前工具仓库 `.gitignore` 忽略 `.expline/`，但需求希望用户实验项目中的 `.expline/` 可随 Git 同步。
- [ ] `expline config get/set` 在未 init 的目录会创建 `.expline/`。
  这与“先 init 再配置”的直觉略有偏差。
- [ ] `run_user_command` 不捕获 stdout/stderr。
  当前符合“不保存完整日志”的边界，但未来可视化页面若需要运行摘要，会缺少日志来源。
- [ ] `diff.patch` 可能保存截断后的 diff。
  后续应区分“完整原始 diff 文件”和“提供给 AI 的截断 diff”。
- [ ] `record.json` 字段与需求示例不完全一致。
  当前实现使用 `evidence_index`，需求示例中写的是 `evidence_files`。
- [ ] `rescan` 会覆盖 `project_summary.md`。
  如果用户手动编辑过项目语义说明，可能造成修改丢失。
- [ ] `rebuild` 已具备基础重建能力，但还没有更完整的链路诊断报告。
  例如孤立节点分组、commit 索引差异报告、修复建议等还未实现。
